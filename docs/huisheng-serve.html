绘声 · TTS-Serve-MLX 后端架构

FastAPI + MLX 语音合成推理引擎

Apple Silicon 本地推理 · 双模型支持 · PyInstaller 打包

基于 FastAPI 构建的高性能语音合成后端，通过 MLX 框架在 Apple Silicon 上运行 Qwen3-TTS 和 VoxCPM2 模型。作为 Electron 子进程运行，提供 RESTful API 服务。

🏗️ 整体架构总览

TTS-Serve-MLX 是一个 FastAPI 应用，通过 uvicorn 启动，作为 Electron 主进程的子进程运行。采用模块化设计，分为 API 层、模型管理层和音频推理层。

🚀 Layer 1 — 启动入口 (server_main.py)

作为独立可执行文件运行，同时适配 Python 直接运行和 PyInstaller 打包模式：

端口发现机制 
# 优先 8000-8050 范围找空闲端口
def _find_free_port() -> int:
    for port in range(8000, 8051):
        with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
            try:
                s.bind(('127.0.0.1', port))
                return port
            except OSError:
                continue
    # 范围内全满，让 OS 分配
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
        s.bind(('127.0.0.1', 0))
        return s.getsockname()[1]

def main():
    port = int(os.environ["TTS_SERVE_PORT"]) if "TTS_SERVE_PORT" in os.environ else _find_free_port()
    print(f"TTS_SERVER_PORT={port}", flush=True)  # ← Electron 主进程抓 stdout 解析
    uvicorn.run("api:app", host=host, port=port, ...)
环境变量配置
变量 说明 默认值
TTS_SERVE_HOST 监听地址 127.0.0.1
TTS_SERVE_PORT 指定端口（不设则自动发现） 8000-8050 自动
TTS_SERVE_MODELS_DIR 模型文件目录 ./models
TTS_SERVE_LOG_LEVEL 日志级别 warning
启动流程
  1. Electron 主进程 spawn tts_serve_mlx 二进制
  2. 后端打印 TTS_SERVER_PORT=xxxxx 到 stdout
  3. Electron 解析端口号，存入 ACTUAL_PORT
  4. Electron 轮询 GET /health（500ms 间隔，120s 超时）
  5. 服务就绪 → Electron 创建窗口

Origin/Referer 安全检查

自定义中间件，在 GPU 推理接口前拦截非本机来源请求，防止恶意网站绕过 CORS 调用接口消耗 GPU 资源：

  • 放行路径：/health、/docs、/openapi.json、/model/download/status（不消耗 GPU）
  • 放行来源：无来源（curl/Electron 主进程）、null（Electron file://）、localhost/127.0.0.1
  • 拒绝：其他来源 → 403
🧠 Layer 3 — 模型管理层 (api.py)
双模型互斥

因为 Apple Silicon 的 GPU 显存有限，两台模型采用互斥加载策略：加载新模型时自动卸载当前的：

@app.post("/model/load")
async def model_load(request: ModelLoadRequest):
    if request.model == "tts":
        unload_vox()       # ← 自动卸载 VoxCPM2
        load_qwen3()       # ← 加载 Qwen3-TTS
    elif request.model == "voxcpm2":
        unload_qwen3()     # ← 自动卸载 Qwen3
        load_vox()         # ← 加载 VoxCPM2
线程安全加载锁 
_model_lock = threading.Lock()

def load_qwen3():
    global _qwen_tts
    with _model_lock:
        if _qwen_tts is not None:
            return  # 已加载，跳过
        instance = TTSClone(model_path=TTS_MODEL_PATH)
        _ = instance.model  # 触发热加载，失败时抛异常
        _qwen_tts = instance

失败安全设计：只有模型完全加载成功后才赋值给全局变量，失败时 _qwen_tts 保持 None。

GPU 内存管理 
def unload_qwen3():
    global _qwen_tts
    with _model_lock:
        if _qwen_tts:
            _qwen_tts.unload()
            _qwen_tts = None
        gc.collect()                # Python GC
        mx.set_cache_limit(0)     # 清空 MLX 缓存
        mx.clear_cache()            # 释放显存

支持 "全杀"模式：不传模型参数时一次性卸载所有模型：

@app.post("/model/unload")
async def model_unload(request: ModelUnloadRequest = None):
    if request is None or request.model is None:
        unload_qwen3()
        unload_vox()
        return {"action": "unloaded", "model": "all"}
按需加载策略

应用启动时不预载任何模型，由前端通过 POST /model/load 触发加载。加载完成后，Electron 主进程自动发送空推理请求预热模型（让 MLX 编译计算图），减少用户首次合成的延迟。

模型信息与下载

内置模型信息字典 _MODEL_SOURCES，包含：

模型 大小 来源 用途
Qwen3-TTS 0.6B (4-bit)~1.6 GBMLX Community语音克隆、批量、对话
VoxCPM2 2B (4-bit)~2.1 GBMLX Community情感克隆、声音设计

模型下载支持双源切换（modelscope / huggingface），异步后台线程下载，通过 GET /model/download/status/{model} 查询进度。

🎵 Layer 4 — 音频管线 (tts_clone.py)
TTSClone 类

基于 mlx_audio.tts.utils.load_model 封装的语音合成引擎：

  • Speaker Embedding 模式：仅传 ref_audio，模型自动提取音色特征
  • ICL 模式：传 ref_audio + ref_text，通过上下文学习还原情感风格，效果更好
批量推理策略

根据输入特点选择最优路径：

  • 原生 batch：所有 items 用同一个 ref_audio + 都传了 ref_text + 不流式 → 走 model.batch_generate()
  • 逐条生成：各 item 用不同 ref_audio 时逐条调用 generate()，带音频读取缓存（避免重复读盘）

VoxCPM2 管线

VoxCPM2 推理结果 48kHz 采样率，支持两个独立端点：

  • /vox/clone — 情感克隆：需要参考音频，支持 ref_text + instruct
  • /vox/design — 声音设计：零样本，仅凭 instruct 描述词（如"御姐声线"）+ 文本生成，无需参考音频

可调参数：inference_timesteps（默认 6，范围 1-10）、cfg_value（默认 4.0，范围 0.5-5.0）

典型请求流程 — 语音克隆
用户点击"合成" → Frontend: ensureModelLoaded('tts') → POST /model/load {model: "tts"} → unload_vox() → load_qwen3() → TTSClone(model_path) → 后台预热: POST /clone {text: '', ref_audio: ''} → Frontend: POST /clone {text, ref_audio} → require_qwen3() → _validate_audio_path(ref_audio) → _qwen_tts.generate(text, ref_audio[, ref_text]) → mlx_audio 推理 → numpy array → save_audio(audio) → /output/clone_xxxx.wav → return {audio_url: "/output/clone_xxxx.wav"} → Frontend: AudioPlayer(getOutputUrl(audio_url))
原始音频返回模式

传 save_file: false 时后端直接返回 WAV 二进制流（media_type="audio/wav"），前端用 arrayBuffer 处理，适合流式或实时处理场景。

📥 Layer 6 — 模型下载系统
异步下载

模型下载在后台线程中运行，不阻塞 API 请求：

  • 请求 POST /model/download → 启动后台线程 → 立即返回

🔒 Layer 7 — 安全设计
路径遍历防护 
def _validate_audio_path(audio_path: str) -> Path:
    raw = Path(audio_path)
    normed = os.path.normpath(audio_path)
    p = Path(normed).resolve()
    if '..' in raw.parts:           # ← 检查原始路径中的 ..
        if not str(p).startswith(str(Path.cwd()) + os.sep):
            raise HTTPException(400, "拒绝访问受保护路径之外的音频文件")
    return p

def _validate_filename(filename: str) -> str:
    if '/' in filename or '\\' in filename or filename.startswith('.'):
        raise HTTPException(400, "无效的文件名")
三层安全模型
  1. 网络层：只监听 127.0.0.1，不暴露到局域网/公网
  2. HTTP 中间件层：Origin/Referer 校验，非本机来源阻止 GPU 推理
  3. 应用层：路径遍历防护 + 文件名校验

build.sh [1/5] 检查 Python 依赖 (fastapi, uvicorn, soundfile, mlx, mlx_audio) [2/5] 检查 libsndfile (自动定位 /opt/homebrew/lib/) [3/5] 检查模型文件（仅验证存在，不打包） [4/5] 清理旧构建 (rm -rf build/ dist/) [5/5] PyInstaller onedir 模式打包 └─ 创建空 models/ 目录 + README 说明 产物结构 dist/tts_serve_mlx/ ├── tts_serve_mlx ← 可执行文件 ├── _internal/ ← Python 运行时 └── models/ ├── qwenTTS_0.6B_MLX/ ← 空壳，需用户放置模型 ├── voxCPM2_4bit_MLX/ ← 空壳，需用户放置模型 └── README.txt
PyInstaller 集成要点
  • 打包前清 PYTHONPATH，防止混入不兼容的包（如其他 venv 的 pydantic_core）
  • 模型文件不打包（体积太大），通过 TTS_SERVE_MODELS_DIR 环境变量指定路径
  • Electron 打包时通过 extraResources 将 tts_serve_mlx 目录嵌入 app
  • 离线版包含模型文件，在线版排除
