# 反馈测试须知

## 🎯 测试目标

验证打包签名修复 + 日志脱敏在生产环境是否正常工作。

---

## ✅ 测试清单

### 1. macOS 15 Gatekeeper 测试（需 M4 用户）

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 下载新包 `huisheng-1.2.1-online-arm64.dmg` | 正常下载 |
| 2 | 打开 DMG，拖到 Applications | 不再报「已损坏，扔垃圾桶」 |
| 3 | 首次打开 | 弹「无法验证开发者」提示 |
| 4 | 去 设置 → 隐私与安全性 → 点击「仍然打开」 | 应用正常启动 |

> 如果还弹「已损坏」→ 说明签名修复不够，需要回炉

### 2. 日志脱敏测试（你自己测）

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 正常使用一会儿（切模型、生成音频等） | - |
| 2 | 点 Mascot → 🐛 → 提交反馈 | Issue 自动创建 |
| 3 | 检查 Issue 日志区 | 绝对路径被裁剪为 `/最后/两段` |

**对照示例：**

```
Before: /Users/coderxu/Library/Application Support/huisheng/models/qwenTTS_0.6B_MLX
After:  /models/qwenTTS_0.6B_MLX

Before: models_dir=/Users/coderxu/Library/Application Support/huisheng/models
After:  models_dir=/huisheng/models
```

> 如果 Issue 里还能看到 `/Users/xxx/` → 脱敏未生效，修

### 3. 签名验证（本地快速检查）

```bash
# 挂载新 DMG
hdiutil attach release/online/huisheng-1.2.1-online-arm64.dmg

# 检查 CodeResources 存在
ls "/Volumes/绘声 1.2.1-arm64/绘声.app/Contents/_CodeSignature/CodeResources"

# 检查签名
codesign -dvvv "/Volumes/绘声 1.2.1-arm64/绘声.app"
# 应看到: Sealed Resources version=2  (不是 none)
#         flags=0x2(adhoc)             (没有 linker-signed)
```

## 🚨 如果出问题

| 现象 | 可能原因 |
|------|----------|
| 仍报「已损坏」 | ad-hoc 签名在 macOS 15 上无论如何不够 → 考虑 $99 Developer Program |
| 日志仍有绝对路径 | 路径不在行末尾或格式特殊 → 看具体 Issue 日志，加 regex 规则 |
| 包打不开（闪退） | Electron 版本/依赖问题 → 跟签名无关，单独排查 |
